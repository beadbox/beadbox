## What's New in v0.17.0

### Added

- **Update notification on startup error screens**: if a newer version of Beadbox is available, the notification now appears even when the app is stuck on the bd-missing or dolt-missing screen. Previously you had to get past startup to see update prompts.

### Changed

- **Server workspace connection via environment variables**: server-only workspaces now connect using bd's native env vars instead of synthesizing runtime directories. Simpler config, fewer moving parts.
- **Minimum bd version raised to 0.58.0**: Beadbox now requires bd 0.58.0+ for DerivePort support. The startup gate will prompt you to upgrade if your bd version is older.
- **"Connect to a server" renamed to "Use existing workspace"**: the button label in the workspace selector now matches what it actually does.
- **WebSocket server consolidated into Next.js**: removed standalone WS server port references. Real-time updates now run through the Next.js server process.
- **Removed Workspace Database section from settings**: the settings dialog no longer shows a database path section that most users never needed.
- **Filter icon tooltips switched to Radix Tooltip**: filter toggle tooltips now appear in ~200ms instead of the browser-default delay.

### Fixed

- **Settings Logs path mislabeled**: the Logs section in Settings showed the `.beads` directory path instead of the actual log file location.
- **Duplicate error overlays**: the amber "failed to refresh" overlay and red health banner could stack on top of each other. Consolidated into a single red health banner.
- **Multi-line error messages truncated**: error banners now render multi-line messages properly, with a "Details" dialog for long error output.
- **Linux: stray browser tab on every launch**: launching Beadbox on Linux no longer opens an extra browser tab alongside the app window.
- **Filter toggle active vs. hover indistinguishable**: active filter icons now have a distinct visual state from hover, so you can tell which filters are on.
- **Non-existent workspace path shows wrong dialog**: entering a path that doesn't exist in AddWorkspaceDialog now shows an inline error instead of opening the InitWorkspaceDialog.
- **Transient "Workspace did not respond" on server workspaces**: connecting to a server workspace for the first time no longer flashes a false error before data loads.
- **Missing left border on depth-0 bead rows**: top-level beads in the epic tree now have the same left border treatment as nested beads.
- **Dependency pill overflow in bead table**: long dependency lists no longer overflow outside their table cell.

### Security

- **DOMPurify updated to 3.3.2**: addresses a known XSS vulnerability in earlier versions.
