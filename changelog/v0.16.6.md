## What's New in v0.16.6

### Added

- **Onboarding architecture redesign**: the startup and onboarding code path has been rewritten. An explicit state machine replaces implicit state scattered across components. Workspace identity is now UUID-based with automatic migration from the old name-based format. Health checks validate workspace connectivity with classified error types. The startup gate is now a proper state machine instead of chained effects.
- **Single-process Tauri sidecar**: the native app no longer spawns a separate WebSocket server process. WebSocket connections route through the Next.js server directly, eliminating port-allocation and process-orphaning bugs across all platforms.
- **WebSocket same-origin architecture**: the WebSocket client connects to the same origin as the web app instead of a separate port. Connection failures use exponential backoff with jitter.
- **Platform-specific error screens**: bd-missing, dolt-missing, and database-error screens now show platform-appropriate install instructions (macOS, Linux, Windows) with a recovery action matrix.
- **"Database not found" recovery UI**: when the workspace database cannot be reached, a recovery screen with a Retry button replaces the blank error state.
- **Server discovery overlap detection**: adding a server whose address resolves to an already-registered local workspace now shows an overlap warning with a Replace option instead of silently creating a duplicate.

### Changed

- **"Add existing workspace" button label**: "Connect to a server" now reads "Use existing workspace," better describing what it does.
- **Add Workspace dialog defaults to Local Path tab**: matches the most common workflow.
- **Filter toggle active state**: active filter icons now use `aria-pressed` styling visually distinct from hover.
- **Filter icon tooltips**: replaced native `title` attributes with Radix UI Tooltips (200ms delay, instant transition between adjacent icons).

### Fixed

- **Windows app DOA on launch**: UNC path corruption in the Tauri sidecar prevented the Windows app from starting. Path handling now preserves UNC prefixes correctly.
- **React dev bundle loaded in production**: `NODE_ENV=production` was missing from the Tauri sidecar environment, so Next.js loaded the development bundle instead of the production build.
- **False "Workspace did not respond" on empty workspaces**: fresh workspaces with no beads triggered a spurious 10-second timeout banner. The health check now distinguishes "empty but healthy" from "actually unreachable."
- **Archiving a bead requires manual refresh**: archiving a bead left it visible in the list until the next refresh. The bead now moves out of view immediately.
- **Drag-and-drop broken on Windows**: drag-and-drop in the bead table failed silently due to a WebView2 OLE compatibility issue. Fixed via a custom MIME type.
- **Duplicate workspace entries in selector**: workspaces could appear twice if registered at both add-time and load-time. Deduplication now runs at both stages.
- **Dolt detection on Linux**: binary detection now probes common Linux install paths that were previously missed.
- **Diagnostics panel false ".beads/ missing" on server workspaces**: the diagnostics panel checked for a local `.beads/` directory even on server-mode workspaces. Detection now respects workspace mode.
- **Server workspace metadata missing server block**: server-mode workspace metadata now includes the `server` block with correct connection info.
- **Workspace removal race condition**: removing a workspace now shows a spinner and disables the button during removal, preventing double-click race conditions.
- **Database not registered after workspace init**: initializing a workspace via the folder picker now registers the database on the running Dolt server immediately.
- **Tauri first-launch blocker**: the startup sequence now ensures the Dolt server is running before presenting the workspace creation flow.
- **Windows: workspace Open clicks silently fail**: fixed a platform-specific path resolution issue that caused Open clicks to do nothing on Windows.
- **Non-existent path opens wrong dialog**: invalid paths in the Add Workspace dialog now show an inline error instead of opening the Init Workspace dialog.
- **bd path resolution returns directory**: `resolveBdDbPath` now returns the full file path instead of the directory path.
